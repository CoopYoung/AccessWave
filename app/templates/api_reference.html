Overview

BASE URL https://your-domain.com

The AccessWave REST API is built with FastAPI and follows standard REST conventions. All request and response bodies are JSON. Dates are ISO 8601 UTC strings.

The interactive Swagger UI lets you authenticate and try every endpoint directly in the browser. A ReDoc view is also available for reading-friendly documentation.

Authentication

JWT Bearer Token

Obtain a token by registering or logging in. Send it on every protected request: 

Authorization: Bearer <your-jwt-token>

API Key

🔑 Create a long-lived API key in your API Keys settings page. Use it exactly like a JWT in the Authorization header. Keys start with aw_. 
Authorization: Bearer aw_a1b2c3d4...

Error responses

Unauthenticated requests receive 401 Unauthorized. Forbidden operations (e.g. plan limits) receive 403 Forbidden.

Rate Limits

Endpoint groupLimit
General API60 requests / minute
POST /api/sites/{site_id}/scan10 requests / minute
POST /api/auth/login10 requests / minute
POST /api/auth/register5 requests / minute

Exceeded limits return 429 Too Many Requests with a Retry-After header indicating when to retry.

Auth Endpoints

POST /api/auth/register Create account → JWT
POST /api/auth/login Log in → JWT
GET /api/auth/me Current user info
PUT /api/auth/profile Update email
PUT /api/auth/password Change password
DELETE /api/auth/account Delete account

Sites

A site is a monitored website. Each site belongs to one user and can have multiple scans.

GET /api/sites List all sites
POST /api/sites Add a site
PATCH /api/sites/{site_id} Update name / URL
DELETE /api/sites/{site_id} Delete site + all scans
PATCH /api/sites/{site_id}/schedule Set scan schedule (none / daily / weekly / monthly)

Scans

Scans run asynchronously. After starting a scan with POST .../scan you can:

  • Poll GET /api/scans/{id} until status is completed or failed.
  • Subscribe to the SSE stream at /api/scans/{id}/stream?token=<jwt> for real-time progress events.
POST /api/sites/{site_id}/scan Start a scan
GET /api/sites/{site_id}/scans List scans (filterable)
GET /api/scans/{scan_id} Get scan details
GET /api/scans/{scan_id}/stream SSE progress stream (token via ?token=)
GET /api/scans/compare?scan_a=&scan_b= Diff two scans

Issues

Issues are individual WCAG violations found during a scan, each with a severity, rule ID, selector, and fix guidance.

GET /api/scans/{scan_id}/issues List issues (filter by severity / rule_id)

Severity levels

critical  serious  moderate  minor

These map directly to the axe-core impact levels.

Sharing

Generate a public, token-authenticated share link for any completed scan. Shared reports are read-only and require no authentication.

POST /api/scans/{scan_id}/share Generate share link
DELETE /api/scans/{scan_id}/share Revoke share link
GET /api/share/{token} Public: get shared scan (no auth)
GET /api/share/{token}/issues Public: get issues for shared scan
GET /api/sites/{site_id}/badge.svg Public: SVG accessibility score badge

API Keys

🔒 The plaintext key is returned only on creation. Store it securely — it cannot be retrieved later.
GET /api/keys List API keys
POST /api/keys Create API key (max 10)
DELETE /api/keys/{key_id} Revoke API key

Webhooks

Register an HTTPS URL to receive scan.completed or scan.failed events. Each delivery is signed with an HMAC-SHA256 signature in the X-AccessWave-Signature header.

GET /api/webhooks List webhooks
POST /api/webhooks Create webhook
PATCH /api/webhooks/{id} Update webhook
DELETE /api/webhooks/{id} Delete webhook
POST /api/webhooks/{id}/test Send test delivery

Backup & Restore

GET /api/backup/export Download all data as JSON
POST /api/backup/import Restore from exported JSON

Health Endpoints

No authentication required. Use these for container liveness / readiness probes.

GET /health Liveness probe (process alive)
GET /ready Readiness probe (DB reachable)

Code Examples

Register and log in (cURL)

# Register
curl -s -X POST https://your-domain.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"s3cr3tpass"}'

# Log in
TOKEN=$(curl -s -X POST https://your-domain.com/api/auth/login \
  -d "username=you@example.com&password=s3cr3tpass" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo $TOKEN

Add a site and start a scan (cURL)

# Add a site
SITE=$(curl -s -X POST https://your-domain.com/api/sites \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Site","url":"https://example.com"}')
SITE_ID=$(echo $SITE | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

# Start a scan
curl -s -X POST https://your-domain.com/api/sites/$SITE_ID/scan \
  -H "Authorization: Bearer $TOKEN"

Retrieve issues (Python)

import requests

BASE = "https://your-domain.com"
TOKEN = "<your-token>"
HEADERS = {"Authorization": f"Bearer {TOKEN}"}

# List critical issues for scan 42
resp = requests.get(
    f"{BASE}/api/scans/42/issues",
    params={"severity": "critical", "limit": 50},
    headers=HEADERS,
)
for issue in resp.json():
    print(issue["rule_id"], "-", issue["message"])

Verify a webhook signature (Python)

import hashlib, hmac

def verify_webhook(payload_bytes: bytes, signature: str, secret: str) -> bool:
    expected = hmac.new(
        secret.encode(), payload_bytes, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)
